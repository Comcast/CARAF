# Contributing

Thank you for your interest in contributing to Xipher CARAF.

## Getting Started

1. Fork the repository and create a feature branch.
2. Set up a local environment and install dependencies.
3. Make focused changes with clear commit messages.
4. Add or update tests where relevant.
5. Open a pull request with a clear description of your changes.

## Development Guidelines

- Follow the existing project structure and coding style.
- Keep changes scoped to the issue being addressed.
- Avoid introducing breaking changes without discussion.
- Ensure security-sensitive code paths are validated.

## Pull Request Checklist

- Code builds and runs locally.
- New or modified behavior is tested.
- Documentation is updated when needed.
- Security and input validation impacts are considered.

## Reporting Security Issues

Please report potential security issues privately through your organization's approved vulnerability disclosure process.
